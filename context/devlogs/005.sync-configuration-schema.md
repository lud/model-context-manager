# Sync configuration schema

Phase 3 of the sync feature plan (spec 002).

Added the sync schema to the project file and resolution logic in `loadProjectOrFail()`.

## What changed

- Added `SyncSpecSchema` and `sync` array field to `ProjectSchema` in `project.ts`
- Added resolved types: `LocalFsUpstream`, `GitHubUpstream`, `ResolvedUpstream`, `ResolvedSyncSpec`
- `ResolvedProject` now includes `sync: ResolvedSyncSpec[]`
- `loadProjectOrFail()` resolves sync specs: string upstream → `localfs` with absolute path, GitHub object → `github` with leading slash trimmed from path, local resolved relative to projectDir
- Default mock updated with `sync: []`
- 11 new tests for parsing and resolution
- New fixture `test/fixtures/doctypes/with-sync/.mcm.json`
- Regenerated `resources/mcm-project.schema.json`
