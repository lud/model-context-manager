# Rename "config" to "project"

Phase 1 of the sync feature plan (spec 002).

Pure mechanical rename — no logic changes. The internal "config" concept is now "project" to better reflect what `.mcm.json` represents.

## What changed

- Renamed `src/lib/config.ts` → `project.ts` (and test + test-helpers)
- Renamed `test/fixtures/config-lookup/` → `project-lookup/`
- Renamed `resources/mcm-config.schema.json` → `mcm-project.schema.json`
- All identifiers updated: `ConfigSchema` → `ProjectSchema`, `Config` → `Project`, `DoctypeConfig` → `DoctypeEntry`, `ResolvedConfig` → `ResolvedProject`, `getConfig` → `getProject`, etc.
- Updated imports and mocks across all 5 command files, their tests, `sequence.ts`, `build-json-schema.ts`, `build-site.ts`
- Updated CLAUDE.md documentation
- `.mcm.json` file name on disk stays unchanged
