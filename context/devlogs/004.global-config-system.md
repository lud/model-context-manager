# Global config system

Phase 2 of the sync feature plan (spec 002).

Added a global config system for storing user-level settings like GitHub tokens, separate from per-project `.mcm.json` files.

## What changed

- New dependency: `env-paths` for XDG-compatible paths
- New `src/lib/global-config.ts` — `getGlobalConfig()`, `saveGlobalConfig()`, `globalConfigPath()`, `normalizeGithubUrl()`
- Config stored at `~/.config/mcm/mcm.global.json` (env-paths returns a directory, file lives inside it)
- Returns defaults when no file exists, Zod-validated with unknown fields stripped, JSON parse errors propagate
- New test file with 11 tests, new test-helper with `mockGlobalConfig()`
