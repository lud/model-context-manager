# GitHub sync

Phase 5 of the sync feature plan (spec 002). Completes the sync feature.

## What changed

- New `src/lib/git.ts` — `cloneRepo()` using `execFileSync` with `stdio: "pipe"` (hides token from output), 60s timeout, returns `{ success, exitCode, stderr }`.
- New `src/lib/git.test.ts` — 6 tests with mocked `child_process`.
- `src/lib/sync.ts` refactored to common path copy architecture:
  - Internal `applyCopy(sourcePath, local, mode)` — shared copy engine for both local and GitHub
  - `syncLocalSpec` now delegates to `applyCopy`
  - New `syncGitHubSpec(spec, token?)` — clones to temp dir, removes `.git`, delegates to `applyCopy`, always cleans up in `finally`
  - Returns `auth_required`, `auth_failed`, or `network_error` in addition to `SyncResult`
- `src/lib/sync.test.ts` — 5 new unit tests for `syncGitHubSpec` (auth_required, auth_failed, network_error, error for wrong upstream kind, skipped when path not in clone).
- `src/commands/sync.ts` — full GitHub flow: token lookup from global config, network error retry (3x), auth_required prompts (with existing-token select or direct password prompt), auth_failed retry/skip flow, token saved to global config. `reportSyncResult` extracted as shared helper.
- `src/commands/sync.test.ts` — 8 new GitHub command tests. Factory mock preserves `normalizeGithubUrl` while mocking `getGlobalConfig`/`saveGlobalConfig`.
- New `src/lib/sync-github.integration.test.ts` — skipped unless `MCM_TEST_GITHUB=1`. Tests public repo (octocat/Hello-World) and private repo (lud/libcheck) using `MCM_TEST_GITHUB_TOKEN`.
