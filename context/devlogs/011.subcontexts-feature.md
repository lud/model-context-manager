# Subcontexts Feature

Added support for "subcontexts" — named feature directories that give a subset of doctypes their own numbering namespace while keeping other doctypes at the project root.

## What changed

### Schema (`src/lib/project.ts`)
- Added `SubcontextsSchema` with `dir` and `doctypes` fields
- Added `subcontexts` optional field to `ProjectSchema`
- Added `ResolvedDoctypeEntry` type with `inSubcontext: boolean`
- Added `rawConfig: Project` to `ResolvedProject` so commands can access unresolved config without re-reading the file
- Added `loadJSONFile(filePath)` for single I/O — `loadRawProject(raw, projectFile, opts?)` is now a pure transform
- Renamed `loadProjectOrFail` → `loadRawProject`; error handling moved to `getProject`
- `getProject` reads the file once, loads base project, optionally reloads with subcontext (both calls co-located)

### Global config (`src/lib/global-config.ts`)
- Added `currentSubcontexts: Record<string, string>` to persist active subcontext per project directory
- Added `getCurrentSubcontext(projectDir)` and `setCurrentSubcontext(projectDir, name)`

### Subcontext helpers (`src/lib/subcontext.ts`)
- `listSubcontexts(dir)` — lists sorted subcontext directory names
- `resolveSubcontextArg(dir, arg)` — resolves a number or fuzzy string to a subcontext dir name
- `nextSubcontextDirName(existingDirs, slug)` — computes the next 000-padded dir name

### `sub` command (`src/commands/sub.ts`)
- `mcm sub add <slug...>` — creates subcontext dir + managed doctype subdirs, auto-selects it
- `mcm sub switch [query]` — switches by number or fuzzy name; shows `@clack/prompts` select when no args
- `mcm sub list` — lists all subcontexts with `*` marking the current one
- `mcm sub current` / `mcm sub which` — prints the path to the active subcontext

### Existing commands
- Added `--sub` flag to `list`, `new`, `next`
- Added guard: if a doctype is managed by subcontexts but no subcontext is active, abort with a clear message

### JSON schema
- Added `subcontexts` property to `resources/mcm-project.schema.json`

## Design notes
- Double `loadRawProject` call only happens when `--sub` is explicitly passed (rare path)
- `rawConfig` on `ResolvedProject` avoids re-reading the file in `subAdd` to get unresolved doctype dirs
- Fuzzy match: strips spaces, builds `.*c1.*c2.*` regex against full dir names (e.g. `001.my-feature`)
