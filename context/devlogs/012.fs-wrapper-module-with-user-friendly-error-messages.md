# fs wrapper module with user-friendly error messages

Added `src/lib/fs.ts` which re-exports `readdirSync`, `readFileSync`, and `writeFileSync` from `node:fs`, each wrapped in a try-catch that calls `abortError` with a human-readable message instead of letting raw Node.js errors surface.

Error messages are context-aware per operation and per error code:

| Code | readdirSync | readFileSync | writeFileSync |
|------|-------------|--------------|---------------|
| ENOENT | directory not found: {path} | file not found: {path} | directory not found: {path} |
| EACCES | permission denied: {path} | permission denied: {path} | permission denied to write: {path} |
| ENOTDIR | not a directory: {path} | — | — |
| EISDIR | — | path is a directory: {path} | path is a directory: {path} |
| other | failed to read directory: {path} | failed to read file: {path} | failed to write file: {path} |

Updated `commands/list.ts`, `commands/new.ts`, and `lib/global-config.ts` to import from the new module.

`commands/next.ts` and `lib/subcontext.ts` were intentionally left on their own error handling since they silently treat a missing directory as empty.
