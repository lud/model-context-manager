# Initial project setup

## What MCM does

MCM is a file management CLI that organizes files by **doctype** — a named category with an associated directory. Each doctype has a configurable filename sequencing scheme (zero-padded counters, datetime stamps, or none).

## Config system (`src/lib/config.ts`)

- `.mcm.json` located by walking up from CWD via `locateConfigFile()`
- Zod-validated schema with `ConfigSchema` (exported for JSON schema generation)
- `parseConfig()` validates raw JSON, strips unknown fields (including `$schema`)
- `loadConfigOrFail()` resolves relative `dir` paths to absolute, returns `ResolvedConfig` with `configFile` and `configDir`
- `getConfig()` aborts with a clear error when no config file is found (no silent empty defaults)
- All Zod validators carry `.describe()` annotations for JSON schema documentation

## Commands

| Command | Type | Purpose |
|---------|------|---------|
| `list [doctype]` | day-to-day | List all doctypes, or files in a specific one |
| `new <doctype> <title...>` | day-to-day | Create a new file with sequenced filename |
| `next <doctype> [title...]` | day-to-day | Preview the next filename without creating it |
| `which` | day-to-day | Print the path to the active `.mcm.json` |
| `demo <name>` | interactive | Demo of `@clack/prompts` and styled output |

All commands have `help.description` for `--help` output. Running `mcm` with no command shows help automatically.

## Supporting libraries

- **`src/lib/cli.ts`** — Output helpers (`writeln`, `warning`, `error`, `abortError`, etc.) using `ansis`. All day-to-day commands use this exclusively.
- **`src/lib/sequence.ts`** — `nextFilename()` computes the next sequenced filename from existing files and doctype config. Supports `"none"`, `"datetime"`, and zero-padded counters (`"000"`, `"0000"`, etc.).
- **`src/lib/slugify.ts`** — Unicode-aware slugification (NFD normalization, diacritics removal, lowercasing).
- **`src/lib/paths.ts`** — `toDisplayPath()` shows relative paths when inside CWD, absolute otherwise.

## JSON schema generation

`tools/build-json-schema.ts` converts the Zod `ConfigSchema` to JSON Schema (draft-2020-12) via `z.toJSONSchema()` with `io: "input"`. Output: `resources/mcm-config.schema.json`. Descriptions from `.describe()` calls propagate into the schema for editor autocompletion.

## Testing

- vitest with 77 tests across 9 test files
- Commands tested by calling `command.callback!()` directly
- Config mocked via `mockConfig()` from `config.test-helpers.ts`
- CLI output captured via `vi.mock("../lib/cli.js")`
- Real fixture files in `test/fixtures/` (no fs mocking, no temp dirs)
- Abort functions mocked to throw so tests stop at the abort point

## Dependencies

- `cleye` for CLI framework
- `@clack/prompts` for interactive commands
- `ansis` for ANSI colors
- `zod` (v4) for config validation and JSON schema generation
