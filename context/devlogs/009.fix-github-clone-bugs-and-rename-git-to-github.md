# Fix GitHub clone bugs + rename git.ts → github.ts

## What changed

### Bug fixes

1. **Double-URL bug**: When `.mcm.json` contains a full GitHub URL like
   `"https://github.com/owner/repo"`, `cloneRepo` would interpolate it into
   `https://github.com/https://github.com/owner/repo.git` — a broken URL.
   Fixed by adding `normalizeGitHubRepo()` in `project.ts` that strips
   `https://github.com/` prefixes and `.git` suffixes during project loading.

2. **Auth format bug**: `cloneRepo` built `https://${token}@github.com/...`
   but git requires `username:token@` format. Fixed to use
   `x-access-token:${token}@`.

### Rename

Renamed `src/lib/git.ts` → `src/lib/github.ts` (and its test file) since it
only contains GitHub-specific code (`cloneRepo` with `github.com` hardcoded).
Updated all imports in `sync.ts` and `sync.test.ts`.

### Tests

Added fixture `test/fixtures/doctypes/github-url-formats/.mcm.json` with four
URL variants and corresponding tests in `project.test.ts` that verify
normalization through the full `loadProjectOrFail` path.

## Why

Live testing revealed these bugs — they weren't caught because tests mocked
each layer with well-formed `"owner/repo"` inputs, so the contract between
project loading and `cloneRepo` was never verified end-to-end.
