# Add seqfix command for renumbering sequenced files

Added `mcm seqfix <doctype>` — a command that fixes sequence numbering in a doctype directory.

## What it does

- Reads all files from the doctype's directory
- Parses numeric prefixes (e.g. `001`, `07`, `0000002`) separated by the doctype's `sequenceSeparator`
- Sorts sequenced files by integer value, breaking ties alphabetically by slug
- Computes new positions (1, 2, 3, …) padded to the doctype's `sequenceScheme` width
- By default prints the planned renames and advises `-f / --force` to apply
- With `--force`, uses a two-phase rename (old → temp → new) to avoid conflicts

## Key rules

- `sequenceScheme` of `"none"` or `"datetime"` → error, unsupported
- `"07"` and `"00007"` are duplicates (both parse as integer 7)
- Files without a numeric prefix are left untouched
- Duplicate sequence numbers are resolved by alphabetical slug order
