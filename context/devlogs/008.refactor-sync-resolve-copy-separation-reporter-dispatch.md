# Refactor sync: resolve/copy separation + reporter + dispatch

Refactored the sync system to cleanly separate source resolution from file copying.

## Changes

**`src/lib/sync.ts`** — Split into resolvers + public `applyCopy`:
- New `resolveLocalSource()` and `resolveGitHubSource()` return a `SourceHandle` (with `ResolvedSource` + optional `cleanup`)
- `applyCopy()` is now public, takes `ResolvedSource` + `SyncReporter` instead of a raw path + `cli.*` calls
- Removed `syncLocalSpec`, `syncGitHubSpec`, `GitHubSyncResult` exports
- Removed `import * as cli from "./cli.js"` — sync library no longer depends on cli output

**`src/commands/sync.ts`** — Dispatch loop with resolver map:
- `resolvers` object keyed by upstream kind replaces if/else dispatch
- Single `applyCopy` call site in the command loop, with `cleanup` in `finally`
- `clackReporter()` creates the `SyncReporter` with clack output
- `resolveGitHubWithAuth()` handles auth/retry logic, returns `SourceHandle | null`

**Tests** — Updated to match new API:
- `sync.test.ts`: Collecting reporter replaces `vi.mock("./cli.js")`; `syncLocal()` helper keeps tests concise
- `sync.test.ts`: GitHub tests use `resolveGitHubSource` + discriminant checks
- `commands/sync.test.ts`: Mocks `resolveLocalSource`, `resolveGitHubSource`, `applyCopy`; verifies `cleanup` is called
- `sync-github.integration.test.ts`: Two-step resolve + applyCopy + cleanup in finally

## Why

The previous design called `applyCopy` from two code paths (local and GitHub), with `sync.ts` importing `cli.js` for output. This made the library hard to test and reuse. The new design has a single `applyCopy` call site, injectable reporting, and clean separation between "resolve the source" and "copy the files."
