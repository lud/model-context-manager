# Site markdown generation build script

Added `tools/build-site.ts` — a build script that generates documentation markdown from CLI command definitions.

## What it does

- Auto-discovers commands in `src/commands/`, imports them, extracts name/description/parameters
- Uses `ts-morph` to extract extended docs from `commentDoc()` JSDoc comments in each command file
- Generates one markdown file per command in `site/commands/` with YAML frontmatter
- Indexes manually-written pages from `site/pages/` (empty for now)
- Outputs `site/index.json` with command and page listings (including `commentDoc` and `description` fields for future `llms.txt` generation)
- Copies JSON schema to `site/assets/`
- Cleans generated directories before each build

## Key design decisions

- `COMMAND_ORDER` controls documentation ordering; `IGNORE_COMMANDS` excludes commands (e.g. `demo`) from docs
- `commentDoc()` is an empty exported function with a JSDoc comment — ts-morph extracts the comment content without running the code
- Generated files (`site/commands/`, `site/assets/`, `site/index.json`) are gitignored; `site/pages/` is committed
- Added `site` recipe to justfile and included it in `check`

## New dependencies

- `ts-morph` (TypeScript AST for comment extraction)
- `yaml` (frontmatter parsing/serialization)
