# Sync Feature — Implementation Plan

## Context

MCM will provide a file synchronization feature to pull files from external sources (local paths, GitHub repos) into projects managed with this tool. This requires several foundational changes: renaming the internal "config" concept to "project" for clarity, introducing a global config system for storing GitHub tokens, and building the sync schema + command.

Each phase is a stopping point for code review. A devlog entry is created after each phase.

## Decisions

- **File on disk**: `.mcm.json` stays unchanged; only internal naming changes
- **Spec sides**: `upstream` (external) / `local` (project-side)
- **Modes**: `receive_merge` (add/update only) / `receive_mirror` (exact copy, deletes extras)
- **Global config**: `env-paths` package for XDG-compatible paths
- **GitHub tests**: Skipped by default, enabled with `MCM_TEST_GITHUB=1` env var, with a console message explaining why tests are skipped
- **Git approach**: Shell out to `git clone --depth 1`

## Phase 2 — Global config system

### New dependency

```
npm install env-paths
```

### New files

**`src/lib/global-config.ts`**

```typescript
export type GlobalConfig = { githubTokens: Record<string, string> }
// Keys: "https://github.com/owner/repo", values: token strings

globalConfigPath(): string          // env-paths("mcm", { suffix: "" }).config + "/mcm.global.json"
getGlobalConfig(): GlobalConfig     // returns defaults if file missing, Zod-validated
saveGlobalConfig(config): void      // creates dir if needed, writes JSON
```

Design:
- `env-paths` returns a directory (`~/.config/mcm/`). We store `mcm.global.json` inside it.
- `getGlobalConfig()` returns `{ githubTokens: {} }` when no file exists (no error)
- `saveGlobalConfig()` creates the directory and file with `mkdirSync({ recursive: true })` if not exist before writing. No need to create anything missing on read operations as we return the default empty config anyway.
- Config is written to `<xdg config dir>/mcm/mcm.global.json`
- Zod schema strips unknown fields (forward-compatible)
- JSON parse errors propagate (corrupted file is not silently ignored)
- When written to global config, Github repos URL must be normalized: `https` always, strip ".git" from URL.

## Phase 3 — Sync configuration schema

### Modify `src/lib/project.ts`

Add to `ProjectSchema`:

```typescript
sync: z.array(SyncSpecSchema).default([])
```

**SyncSpec schema** (as written in `.mcm.json`):

```typescript
{
  upstream: string | { github: "owner/repo", path: "path/in/repo" }
  local: string          // relative to projectDir or absolute
  mode: "receive_merge" | "receive_mirror"   // default: "receive_merge"
}
```

**Resolved types** (after parsing):

```typescript
type LocalFsUpstream  = { kind: "localfs"; path: string }   // path is absolute
type GitHubUpstream   = { kind: "github"; repo: string; path: string }
type ResolvedUpstream = LocalFsUpstream | GitHubUpstream

type ResolvedSyncSpec = {
  upstream: ResolvedUpstream
  local: string   // absolute path
  mode: "receive_merge" | "receive_mirror"
}
```

**Resolution in `loadProjectOrFail()`:**
- String upstream → `{ kind: "localfs", path: resolve(projectDir, upstream) }`
- GitHub upstream → `{ kind: "github", repo, path }` with leading slash trimmed from path
- `local` resolved relative to `projectDir` (same as doctype `dir`)

## Phase 4 — Local filesystem sync

### New files

**`src/lib/sync.ts`** — Pure sync logic, no CLI output.

```typescript
type SyncResult =
  | { status: "synced" }
  | { status: "skipped"; reason: string }
  | { status: "deleted_target" }
  | { status: "error"; message: string }

syncLocalSpec(spec: ResolvedSyncSpec): SyncResult
```

Synchronization rules:
1. Source doesn't exist + `receive_merge` → `skipped` (warning)
2. Source doesn't exist + `receive_mirror` + target exists → delete target, `deleted_target`
3. Source doesn't exist + `receive_mirror` + no target → `skipped`
4. Source is dir, target is file → `error` (caller aborts)
5. Source is file → copy to local (create parent dirs), `synced`
6. Source is dir, target doesn't exist → create dir, copy contents, `synced`
7. Source is dir, target is dir, `receive_merge` → copy all from source into target recursively (no deletes)
8. Source is dir, target is dir, `receive_mirror` → copy all + delete entries in target not in source

Key: directory sync operates on *contents*, not the directory itself (no nesting).

All files operations must be printed with writeln/success/warning from cli module. (`success` for copied files, `warning` for deleted files.)

**`src/lib/sync.test.ts`** — Tests with real files in temp directories.

All 8 rules above tested, plus:

- Empty source directory
- Parent directory creation for file targets

## Phase 5 — GitHub sync

### New file

**`src/lib/git.ts`** — Isolated git operations.

```typescript
type CloneResult = { success: true; dir: string } | { success: false; exitCode: number }

cloneRepo(repo: string, targetDir: string, token?: string): CloneResult
```

- Uses `execSync("git clone --depth 1 ...")` with `stdio: "pipe"` (suppresses output, hides token)
- Token embedded in URL: `https://<token>@github.com/owner/repo.git`
- 60s timeout
- Exit code 128 = failure (repo not found or auth error)
- Always use main branch for now.

**`src/lib/git.test.ts`** — Mock `child_process.execSync`.

### Add to `src/lib/sync.ts`

```typescript
type GitHubSyncResult = SyncResult
  | { status: "auth_required"; repo: string }
  | { status: "auth_failed"; repo: string }

syncGitHubSpec(spec: ResolvedSyncSpec, token?: string): GitHubSyncResult
```

- Clones to temp dir, resolves `upstream.path` within it
- Removes `.git` directory from clone before syncing (prevent syncing git internals)
- Delegates to `syncLocalSpec` for actual file operations
- Cleans up temp dir in `finally` block

### Update `src/commands/sync.ts`

Replace GitHub placeholder with full flow:

1. Load global config, check for existing token for this repo
2. Call `syncGitHubSpec(spec, existingToken)`
3. If `auth_required`:
   - If other tokens in global config → show `@clack/prompts` select (repo names + "Enter new token")
   - Else → prompt directly for new token
   - Show link: `https://github.com/settings/personal-access-tokens/new` with instruction to grant "Contents" read access to the repo
   - Save token to global config, even if using other repo token from clack prompt.
   - Retry `syncGitHubSpec(spec, token)`
4. If `auth_failed` → prompt for "Retry with another token" (then loop back to prompt token with link display) or "Skip" and move to next spec.
5. Other results → report as before
6. Pure network errors must be retried (as long as we can determine it is actually a network error).

### Common path copy

We shall use the same function for all files copies + logs that will create/update/delete files to the local. So the architecture is to have

1. A function that acts as the top level logic: calls the upstream source, then call the copy function.
2. Upstream source: A function that obtains files from the upstream (locate locals files, and for github the function clones the repo and on success locates the files as well). It returns an array of local files (from the filesystem, possibly from a cloned repo in temp dir).
3. Copy function: iterates over the upstream array and does the copies and logs.

This way we can add more sources easily as long as they return an array of files.
